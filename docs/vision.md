# peak vision

## Product summary

`peak` is a one-page Solana project that turns holding into a live community system.

The project should feel:

- simple for non-technical users
- transparent enough to build trust quickly
- professional enough to look like real work
- fun without becoming chaotic

## Core positioning

`peak` is not trying to win by promising endless utility.

It wins by doing a few things clearly:

- showing live holder information
- running transparent reward rounds
- publishing the rules before rewards happen
- proving that the team is building in public

## Experience goals

The website should communicate the project in one pass. A new visitor should quickly understand:

1. What `peak` is
2. Why holding matters
3. How rewards work
4. How the reserve is funded
5. Why the project looks trustworthy

## Product tone

The tone should feel:

- calm
- technical
- confident
- slightly playful

It should not feel:

- corporate
- unserious
- loud for no reason
- dependent on hype language

## V1 scope

Version 1 should stay narrow on purpose.

Included:

- single-page app
- holder leaderboard
- reward explanation
- treasury / reserve transparency section
- next snapshot / next round visibility
- public documentation

Not included:

- too many pages
- complex point systems
- too many reward categories
- manual team-picked rewards
- features that are difficult to explain in one screen

## Trust model

To feel legitimate, `peak` should make key things visible:

- reward rules
- reserve wallet purpose
- dev wallet policy
- prior payouts
- active development

The public repo is part of the product. It should make the project feel deliberate and auditable.
