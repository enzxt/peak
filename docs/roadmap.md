# peak roadmap

## Build strategy

`peak` should be built in public and in stages.

The goal is not to launch every possible idea. The goal is to launch a small system that works, prove that the rules are sustainable, and expand carefully.

## Phase 0: Foundation

Objective:
Lock the public design before implementation.

Deliverables:

- project vision
- tokenomics draft
- reward logic specification
- dev wallet / reserve policy
- public repo structure

Why this phase matters:

This is where trust starts. If the project looks thought-through before launch, that alone becomes part of the product.

## Phase 1: V1 system design

Objective:
Define exactly how v1 rewards operate.

Deliverables:

- reward reserve model
- eligibility rules
- snapshot schedule
- reward round definitions
- transparency requirements

Exit condition:

The full v1 reward system can be explained simply and audited by a normal holder.

## Phase 2: Initial implementation

Objective:
Build only the minimum product needed to operate v1.

Deliverables:

- single-page app
- live stats section
- leaderboard module
- reward information module
- reserve transparency section
- initial backend/indexing endpoints

Exit condition:

A user can understand the project and verify the current reward state from one page.

## Phase 3: Launch operations

Objective:
Run the first rounds conservatively and prove consistency.

Deliverables:

- published first reward rounds
- first snapshot schedule
- first public distribution records
- social launch messaging aligned with public docs

Exit condition:

The system has completed early rounds without confusing holders or stressing the reserve.

## Phase 4: Measured expansion

Objective:
Add new mechanics only after v1 proves healthy.

Possible additions:

- additional leaderboard categories
- improved reward history views
- broader automation
- public toolkit/API positioning
- partner/community events

Rule:

No new feature should ship unless it improves retention, trust, or clarity.

## Public messaging themes

Before launch, the public story should emphasize:

- small scope done well
- transparent rules
- conservative early rounds
- active development in public

This helps position `peak` as a durable project rather than a short-lived hype cycle.
